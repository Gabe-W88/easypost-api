## üìÑ Documentation Standards

### For Audits:
- Use `AUDIT_TEMPLATE.md` as the base for all new audits
- File naming: `AUDIT_TYPE_YYYYMMDD.md` (e.g., `PERFORMANCE_AUDIT_20251203.md`)
- Include: Executive Summary, Methodology, Findings (Critical/High/Medium/Low), Recommendations, Action Items

### For Guides:
- Keep technical documentation current with codebase changes
- Update system documentation when architecture changes
- Include code examples and troubleshooting steps

### For Planning:
- Create planning docs for major initiatives
- Archive or delete planning docs once implemented
- Keep only active/future plans in this folder
